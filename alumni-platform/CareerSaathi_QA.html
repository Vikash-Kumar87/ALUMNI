🎓 CareerSaathi

Intelligent Platform to Bridge Alumni & Students

📋 Project Q&A — Interview / Presentation Sheet
1. Project Overview
1
What is your project?
CareerSaathi is a full-stack web platform that connects college alumni and students through an intelligent bridge. It includes features like AI-powered mentorship matching, a career guidance chatbot, interview practice, a job board, and a discussion forum.
2
Why did you build this project?
Students often struggle to find senior alumni for career guidance. There is no dedicated platform for job referrals, mock interviews, and skill roadmaps. CareerSaathi solves all these problems in one place with AI assistance, available 24/7.
3
Who will use this project?
  • Students — For career guidance, interview practice, job search, and finding mentors
  • Alumni — For posting jobs, providing mentorship, and contributing to the community
  • Admin — For managing the platform and verifying users
  • Colleges / Universities — For digitizing their alumni network
4
What main problem does this project solve?
  • Students lack proper career guidance — the AI chatbot is available 24/7
  • Alumni networks remain disconnected — the platform keeps everyone connected
  • No dedicated mock interview platform — AI-generated questions and evaluation are available
  • Students can't connect for job referrals — an integrated job board is available

10
How does the Career Chatbot work?
User sends a message → Backend verifies the Firebase token → Message is sent to the Groq API (LLaMA 3.3 70B) with conversation history → AI response is received → Displayed on the frontend. The complete conversation history is maintained for multi-turn chat.
11
How does the Interview Practice feature work?
  • Student chooses a topic (e.g., React, Python) and difficulty (easy/medium/hard)
  • AI generates 5 questions in JSON format
  • Student types the answer for each question
  • AI evaluates each answer — providing a score (0–10), strengths, weaknesses, and a model answer
  • An overall performance summary is given at the end
12
What is the Skill Roadmap feature?
The student types any skill (e.g., "Machine Learning", "Web Development") and AI generates a step-by-step personalized learning roadmap — with resources, timeline, and priority. The roadmap is also saved on the dashboard.
4. Architecture & Security
13
What is the project architecture?
3-tier architecture:

Tier 1 — Frontend (Vercel): React app — UI, routing, state management
Tier 2 — Backend (Render): Express API — business logic, AI calls, security middleware
Tier 3 — Database (Firebase): Firestore (structured data) + Realtime DB (chat)

Frontend → Backend REST API → Firebase DB + Groq AI
14
What have you done for security?
  • Firebase Authentication — JWT token is verified with every request
  • Helmet.js — HTTP security headers are automatically set
  • Rate Limiting — 500 requests per 15 min per user; 50 req/min for AI
  • CORS — only the allowed frontend URL can access the backend
  • Environment Variables — no secret keys are hardcoded in the code
  • Firestore Security Rules — users can only read/write their own data
15
How does authentication work?
User logs in (Google or Email/Password) → Firebase Authentication provides a JWT token → Frontend sends this token in the Authorization header with every API call → Backend verifyToken middleware verifies the token using Firebase Admin SDK → only then is the data returned.
16
How is role-based access implemented?
During signup, the user chooses their role: Student / Alumni / Admin. This role is stored in the user document in Firebase Firestore. The Admin panel is visible only to users with the Admin role. Alumni can post jobs; students cannot.

6. AI / Technical Deep Dive
20

Groq was chosen because it is the fastest and provides high limits in the free tier.
21
What is the LLaMA 3.3 70B model?
LLaMA (Large Language Model Meta AI) is Meta's open-source AI model. 70B means 70 billion parameters — it is comparable to ChatGPT in intelligence. The "Versatile" variant is optimized for general-purpose tasks — coding, writing, analysis, and question answering.
22
How is real-time chat implemented?
Firebase Realtime Database is used. When a user sends a message, it is saved in Firebase at the /chats/{chatId}/messages path. Firebase's onValue listener on both users' frontends listens for real-time updates — as soon as a message arrives, it is instantly shown on screen without a page refresh.
23
Why use TypeScript instead of JavaScript?
TypeScript provides type safety — runtime errors are caught at compile time. Bugs are reduced in large codebases, better autocomplete is available in the IDE, and team collaboration becomes easier.
